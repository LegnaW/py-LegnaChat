# Skill创建简明指南
## 注意：这是一个基础的Skill创建指南，由开发者编写，是十分重要的基本参考文件。所以如非必须，请不要乱动这里面的内容！
## 一、基本结构（必需）
```
skill/[技巧名]/
├── display.txt      # 简短标题（单行）
└── description.md   # 详细说明
```

## 二、文件要求

### 1. display.txt
- 单行文本
- 简洁明了（<20字）
- 准确描述功能
- **示例**：`网络爬虫`、`应用列表获取`

### 2. description.md
- Markdown格式
- 包含以下部分：
  ```
  # 标题
  ## 概述（简要介绍）
  ## 使用方法（核心命令/代码）
  ## 示例（实际用例）
  ## 注意事项（重要提醒）
  ```

## 三、创建流程

1. **明确需求**：要解决什么问题？
2. **创建文件夹**：`skill/技巧名`
3. **写display.txt**：简短标题
4. **写description.md**：详细说明
5. **测试验证**：确保功能正常
6. **检查清单**：完成度检查

## 四、命名规范

### 文件夹命名：
- 英文小写
- 下划线分隔
- 简短描述
- **示例**：`web_crawler`、`file_tools`

### 避免：
- 中文名称
- 过长名称
- 无意义名称

## 五、质量要求

### 文档质量：
- ✅ 准确无误
- ✅ 结构清晰
- ✅ 示例充分
- ✅ 注意事项明确

### 功能质量：
- ✅ 正确实现
- ✅ 稳定可靠
- ✅ 使用简单
- ✅ 安全无害

## 六、安全底线

### 绝对禁止：
- ❌ 破坏用户文件
- ❌ 窃取隐私信息
- ❌ 执行未授权操作
- ❌ 隐藏恶意功能

### 必须做到：
- ✅ 用户知情同意
- ✅ 操作可逆
- ✅ 错误安全处理
- ✅ 最小权限原则

## 七、快速检查清单

创建完成后检查：
```
[ ] 1. display.txt是否简洁准确？
[ ] 2. description.md是否有概述、用法、示例、注意事项？
[ ] 3. 功能是否测试通过？
[ ] 4. 命名是否符合规范？
[ ] 5. 安全风险是否已考虑？
```

## 八、最佳实践

1. **简单至上**：功能单一明确
2. **文档先行**：先写文档再实现
3. **用户中心**：考虑使用场景
4. **测试驱动**：边开发边测试
5. **持续改进**：根据反馈优化
